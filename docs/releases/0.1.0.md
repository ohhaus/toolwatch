# ToolWatch 0.1.0

ToolWatch v0.1.0 is an experimental developer tool and portfolio project. It
demonstrates deterministic runtime controls and observability for AI-agent tool calls;
it is not intended for public production deployment.

## Highlights

- Trusted registry and bounded agent sessions.
- Deterministic validation, redaction, risk classification, and block/flag/allow rules.
- PostgreSQL idempotency, audit history, traces, metrics, dashboard, and Attack Lab.
- Optional local Ollama loop; automated tests use the deterministic fake provider.
- Conservative recovery for stale calls/runs and bounded graceful shutdown.
- Wheel, sdist, OCI image, SPDX SBOMs, and GitHub artifact attestations.

## Install and run

```bash
uv sync --frozen
make infra-up
make migrate
make seed
make run
```

Build distributable artifacts with `make package-check`, image with `make image`, and
SBOMs with `make sbom`.

## Architecture and security

The modular monolith keeps the domain independent of FastAPI, SQLAlchemy, HTTPX, and
OpenTelemetry. Every model-selected call resolves through the trusted registry and the
same deterministic security pipeline. A blocked call stops before adapter execution.

Recovery uses PostgreSQL `FOR UPDATE SKIP LOCKED` and short transactions. A stale
`executing` call becomes `failed/execution_state_unknown`; ToolWatch never guesses that
an unknown side effect succeeded and never retries it automatically.

## Demo

Run `scripts/demo_v010.sh`; add `--ollama` for the optional local model and `--cleanup`
to stop infrastructure afterward.

## Provenance verification

Download the release artifacts and GitHub attestation, then run:

```bash
gh attestation verify toolwatch-0.1.0-py3-none-any.whl --repo OWNER/REPOSITORY
```

SPDX files describe the locked Python components and release image reference.

## Upgrade path

This is the first release. Apply `uv run alembic upgrade head` before starting v0.1.0.
Downgrades are supported for development verification but should be preceded by backups.

## Known limitations and roadmap

There is no authentication, public-dashboard safety, production multi-tenancy, approval
workflow, streaming, MCP, cloud provider, RAG, memory, or real downstream integration.
Cancellation is cooperative and cannot roll back an external side effect. Future work
should begin with authentication and deployment isolation, not broader autonomous power.
